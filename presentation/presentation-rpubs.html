Intro

This presentation acts as part of the completion requirement of the Coursera John Hopkins Data Science Specialisation Capstone.

The capstone project is about building a word prediction app with R. The app takes the input of word(s)/sentences, and generates a list of predictions of the next word as output.

The presentation is separeted as the following sections:

  • A general introduction to the app.

Instructions and Functions of the App

The app is built on top of Shiny, a web application framework for R.

  • When first loading the page, nothing except the text input box is shown. The user just need to type the word(s)/sentence he/she wants to predict from.
  • Then, 3 sections will show up:
    • A prediction ranking graph: Up to 15 predictions are sorted by probability.
    • A prediction list: Up to 5 top ranked predictions are shown. Once the user pressed the tick button, the corresponding prediction will be attached to the text input box at the top.
    • A wordcloud: Up to 15 predictions are shown in a format of word cloud (same colour indictes same probability)
  • Changing the text input will be reflected immediately in the prediction details.

The Algorithm

1. Data Preprocessing (Cleaning the training data)

  • Decoding and Translation.
  • Removal of numbser, punctuations, redundant spaces, and lowercase the files.
  • N-gramming (1 / 2 / 3 / 4 grams) and bulid a probability table for each gram.

2. Modelling and Predicting (Applying a simple model)

  • Search from a higher gram table, if found, then sort the predictions by probability, if not, go to a lower gram table, until reaching the uni-gram (Simple back-off model).
  • Build a Kneser-Ney model, considering all the uni-gram words by combining the proability in each gram level.
  • Having compared the results from above two, the accuracy is similar but the simple back-off has a much much better runtime performance. Therefore, the simple back-off is being applied.
